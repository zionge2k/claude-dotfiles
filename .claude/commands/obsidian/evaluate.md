# SLIPBOX Evaluation - $ARGUMENTS

**모드 전환: 구현 → 평가.** 이 커맨드는 SLIPBOX 노트가 단순 요약이 아닌 진짜 통찰을 담고 있는지 비판적으로 검토합니다.

## 작업 프로세스

1. `$ARGUMENTS`로 전달된 SLIPBOX 파일을 읽는다
2. 참고 문서(`[[wikilink]]`로 연결된 RESOURCES 문서)를 함께 읽는다
3. 평가 기준을 적용한다
4. 평가 리포트를 출력한다

$ARGUMENTS가 제공되지 않으면, `000-SLIPBOX/`에서 가장 최근 수정된 파일을 대상으로 한다.

## 평가 기준

### 1. 고유 통찰 (Originality)

SLIPBOX의 핵심 가치는 **"남의 글 요약"이 아니라 "나의 생각"**이다.

| 검증 항목 | 판정 |
|----------|------|
| 참고 문서에 없는 자신만의 주장/관점이 있는가 | 없으면 ❌ |
| 여러 출처를 교차하여 새로운 연결을 만들었는가 | 1개 출처만 있으면 ⚠️ |
| "~라고 한다" 서술만 있고 "나는 ~라고 생각한다"가 없는가 | 전자만 있으면 ❌ |

### 2. 참고 연결 (Grounding)

통찰은 근거가 있어야 한다. 개인 의견과 출처 기반 주장이 구분되어야 한다.

| 검증 항목 | 판정 |
|----------|------|
| RESOURCES 문서에 대한 `[[wikilink]]` 참조가 있는가 | 없으면 ⚠️ |
| 참고 문서의 핵심 내용을 정확히 인용/참조하는가 | 왜곡이 있으면 ❌ |
| 참고 문서에서 도출한 근거와 자신의 해석이 구분되는가 | 섞여 있으면 ⚠️ |

### 3. 원자성 (Atomicity)

하나의 SLIPBOX = 하나의 핵심 주장. Zettelkasten의 기본 원칙.

| 검증 항목 | 판정 |
|----------|------|
| 하나의 핵심 주장/통찰에 집중하는가 | 여러 주제가 섞이면 ⚠️ |
| 제목이 핵심 주장을 한 문장으로 표현하는가 | 모호하면 ⚠️ |
| 다른 SLIPBOX 노트로 분리해야 할 내용이 포함되어 있는가 | 있으면 분리 제안 |

### 4. 반박 가능성 (Falsifiability)

좋은 통찰은 반박할 수 있다. "모두가 동의하는 당연한 말"은 통찰이 아니다.

| 검증 항목 | 판정 |
|----------|------|
| 누군가가 합리적으로 반대할 수 있는 주장인가 | "당연한 말"이면 ⚠️ |
| 구체적 조건/맥락이 명시되어 있는가 | "항상", "모든" 같은 절대 표현이면 ⚠️ |
| 적용 범위의 한계를 인식하고 있는가 | 한계 언급 없으면 ⚠️ |

## 출력 형식

```markdown
## SLIPBOX 평가: {파일명}

**종합**: ✅ 통과 | ⚠️ 보완 필요 | ❌ 재작성 권장

| 기준 | 상태 | 비고 |
|------|------|------|
| 고유 통찰 | ✅/⚠️/❌ | ... |
| 참고 연결 | ✅/⚠️/❌ | ... |
| 원자성 | ✅/⚠️/❌ | ... |
| 반박 가능성 | ✅/⚠️/❌ | ... |

### 개선 제안
1. ...

### 분리 제안 (해당 시)
- "X 부분은 별도 SLIPBOX로 분리하면 좋겠습니다: {제안 제목}"
```
