# 강의 준비 파이프라인 스킬

## 개요

Obsidian vault에 축적된 지식을 기반으로 **강의/코칭 자료를 체계적으로 준비**하는 파이프라인 스킬.
기존 도구들(vis, zettelkasten-expert, create-presentation)을 순차적으로 조합하여 실행한다.

## 핵심 아키텍처

```
사용자 입력: "TDD 강의 준비해줘"
         │
         ▼
┌─────────────────────────────────────────┐
│  Phase 1: 지식 수집 (vis collect)        │
│  - vis collect "주제" --top-k 30         │
│  - 수집된 문서 목록 확인                    │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  Phase 2: 핵심 개념 추출                  │
│  - 수집된 문서들에서 핵심 개념 분석           │
│  - 개념 간 관계 및 학습 순서 도출            │
│  - 코드 예제 및 실습 자료 식별              │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  Phase 3: 강의 구조 설계                  │
│  - 대상 청중 수준에 맞는 구조 제안           │
│  - 시간 배분 계획                         │
│  - 각 섹션별 핵심 메시지 정리               │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  Phase 4: 결과물 생성                     │
│  - /obsidian:create-presentation 호출    │
│  - 또는 마크다운 강의노트 생성              │
│  - vault에 저장                          │
└─────────────────────────────────────────┘
```

## 실행 프로세스

### Phase 1: 지식 수집

1. 사용자가 제공한 주제로 `vis collect` 실행
   ```bash
   vis collect "주제" --top-k 30 --expand
   ```
2. 수집 결과에서 문서 목록과 관련도 확인
3. 관련도 높은 상위 10~15개 문서를 선별
4. 필요 시 `vis search`로 추가 검색

### Phase 2: 핵심 개념 추출

1. 선별된 문서들을 분석하여 **핵심 개념 목록** 작성
2. 각 개념에 대해:
   - 정의 (한국어 + 영어 원문)
   - 관련 코드 예제
   - 다른 개념과의 관계
3. 개념 간 **의존 관계 그래프** 작성 (어떤 개념을 먼저 설명해야 하는지)
4. vault에 부족한 내용이 발견되면 사용자에게 알림

### Phase 3: 강의 구조 설계

사용자에게 다음을 질문하여 구조를 결정:

| 질문 | 기본값 |
|------|--------|
| 대상 청중 수준 | 중급 개발자 |
| 강의 시간 | 60분 |
| 형식 | 슬라이드 + 실습 |
| 코드 예제 언어 | Java |

결정된 조건에 맞춰:
1. 섹션 분할 (도입-본론-실습-정리)
2. 각 섹션 시간 배분
3. 핵심 메시지 1줄 정리
4. 실습 시나리오 설계

### Phase 4: 결과물 생성

사용자 선택에 따라 다음 중 하나를 생성:

| 결과물 | 도구 | 저장 위치 |
|--------|------|-----------|
| 프레젠테이션 슬라이드 | `/obsidian:create-presentation` | vault 내 |
| 강의 노트 (마크다운) | 직접 생성 | `000-SLIPBOX/` |
| 실습 가이드 | 직접 생성 | `003-RESOURCES/` |

## 사용 예시

```
# 기본 사용

사용자: "Hexagonal Architecture 강의 준비해줘. 대상은 주니어 개발자, 90분, Java 코드 예제 포함"

# 프레젠테이션 요청
사용자: "DDD 강의 슬라이드 만들어줘"
```

## 주의사항

- vault에 축적된 지식이 부족한 주제는 **부족한 영역을 먼저 알려주고** 추가 학습을 제안한다
- 강의 구조 설계 시 사용자와 **대화형으로 조건을 확인**한다 (자동 결정하지 않음)
- 코드 예제는 vault에 있는 것을 우선 활용하고, 없으면 새로 작성한다
